# 802.11b Protocol Deep Dive

## Background

802.11b was the first widely deployed Wi-Fi standard, operating in the 2.4 GHz ISM band. It uses HR/DSSS (High Rate Direct Sequence Spread Spectrum) technology, supporting four data rates: 1, 2, 5.5, and 11 Mbps.

## Technical Approach

- **Modulation**: DBPSK (1 Mbps) / DQPSK (2 Mbps) / CCK (5.5/11 Mbps)
- **Spreading**: Barker code (1/2 Mbps) / CCK codewords (5.5/11 Mbps)
- **Channel bandwidth**: 22 MHz
- **Channel plan**: 14 channels (5 MHz spacing), only 3 non-overlapping

## Protocol Analysis

### PPDU Frame Structure

802.11b defines two PPDU formats: Long Preamble and Short Preamble. The main fields include:
- SYNC (synchronization)
- SFD (Start Frame Delimiter)
- SIGNAL (rate/length)
- SERVICE
- PSDU

### Key Technical Points

1. **CCK Modulation**: 5.5 Mbps uses 4 CCK codewords (4 bits each), 11 Mbps uses 64 CCK codewords (8 bits each). CCK codewords are selected from a complementary code keying set with the cross-correlation property:

   $$ R_{ij}(\tau) = \sum_{k=0}^{N-1} c_i[k] \cdot c_j^*[k+\tau] \approx 0 \quad (i \neq j) $$

   where $c_i$ denotes the $i$-th CCK codeword and $N=8$ is the codeword length.

2. **Processing Gain**: The DSSS processing gain $G_p$ is the ratio of the spreading chip rate $R_c$ to the data rate $R_b$:

   $$ G_p = 10 \log_{10}\!\left(\frac{R_c}{R_b}\right) \ \text{[dB]} $$

   For the 1 Mbps mode (Barker code, $R_c = 11\ \text{Mcps}$), $G_p \approx 10.4\ \text{dB}$.

3. **Receiver Sensitivity**: 11 Mbps requires $-76\ \text{dBm}$, 1 Mbps requires $-94\ \text{dBm}$
4. **Carrier Frequency Tolerance**: $\pm 25\ \text{ppm}$
